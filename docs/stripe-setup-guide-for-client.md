# ☑️ Инструкция для заказчика: как настроить Stripe

> 15–20 минут и ваш аккаунт Stripe полностью готов к работе с нашим проектом. Просто выполните шаги ниже по порядку — технические знания не требуются.

---

## 1. Создайте (или войдите в) аккаунт Stripe

1. Откройте сайт <https://dashboard.stripe.com/>.
2. Нажмите **Sign up** (или **Sign in** если у вас уже есть аккаунт).
3. Подтвердите E-mail, перейдя по ссылке из письма.
4. После входа убедитесь, что в левом верхнем углу включён оранжевый переключатель **Test mode** — мы сначала работаем в тестовом режиме, деньги списываться не будут.

> **Если у вас появится кнопка «Activate payments» — можете пропустить её сейчас.** Для тестов она не нужна.

---

## 2. Получите два API-ключа

1. В левой колонке нажмите **Developers → API keys**.
2. Секция **Standard keys** показывает два ключа:
   - **Publishable key** (начинается с `pk_test_…`).
   - **Secret key** (начинается с `sk_test_…`). Нажмите **Reveal key** и скопируйте.

> Эти ключи нужны нашей команде, чтобы подключить оплату.
> • **Publishable key** — можно отправить в чате.  
> • **Secret key** — передайте только через безопасный способ (см. шаг 6).

---

## 3. Создайте продукты и цены

1. В меню слева выберите **Products → + Add product**.
2. Заполните:
   • **Name** — «AI Credits 250 / Month».  
   • **Description** — подписка на 250 кредитов каждый месяц.
3. Нажмите **Save product**.
4. На странице продукта нажмите **+ Add price** и заполните:
   - **Price** → `9.99`.
   - **Billing period** → **Recurring → Monthly**.
   - Нажмите **Add price**.
5. Скопируйте **Price ID** этого плана (будет нужен как «Subscription ID»).

### Одноразовые пакеты (top-up)

Создайте **три** отдельные позиции:

| Пакет  | Name в Stripe           | Price   | Credits        |
| ------ | ----------------------- | ------- | -------------- |
| Small  | **100 Credits Top-up**  | `4.99`  | +100 кредитов  |
| Medium | **500 Credits Top-up**  | `19.99` | +500 кредитов  |
| Large  | **1500 Credits Top-up** | `49.99` | +1500 кредитов |

Для каждого пакета:

1. **Products → + Add product** → заполните _Name_ (из таблицы) и, по желанию, description.
2. Нажмите **Save product**.
3. **+ Add price**:
   - **Price** → сумма из таблицы.
   - **Billing period** → **One time**.
   - Сохраните и скопируйте **Price ID**.

В итоге у вас будет **четыре Price ID** (1 подписка + 3 top-up).

---

## 4. Создайте Webhook (нужен для уведомлений Stripe → наш сервер)

1. Опять **Developers → Webhooks → + Add endpoint**.
2. В поле **Endpoint URL** вставьте адрес, который мы вам пришлём.  
   (Если ещё не получили — пока пропустите, вернётесь позже.)
3. В списке **Events to send** отметьте:
   - `checkout.session.completed`
   - `invoice.paid`
   - `invoice.payment_failed`
4. Нажмите **Add endpoint**.
5. Stripe покажет поле **Signing secret** (начинается с `whsec_…`). Нажмите «Reveal» и сохраните значение.

---

## 5. Итоговый список данных, которые нужно передать нам

| Что                                   | Где вы его взяли              | Как передать                      |
| ------------------------------------- | ----------------------------- | --------------------------------- |
| **Publishable key** `pk_test_…`       | Developers → API keys         | Можно в обычном чате (не секрет). |
| **Secret key** `sk_test_…`            | Developers → API keys         | **Только безопасно** (см. ниже).  |
| **Price ID (Top-up)** `price_…`       | Страница цены пакета          | В чате или файле.                 |
| **Price ID (Subscription)** `price_…` | Страница цены подписки        | В чате или файле.                 |
| **Webhook secret** `whsec_…`          | Страница созданного Webhook'a | Безопасно (см. ниже).             |

---

## 6. Как отправить секретные данные безопасно

1. **Менеджер паролей** (1Password, LastPass) — создайте элемент «Secure Note» и поделитесь ссылкой.
2. **Защищённый файл** — ZIP/7-Zip с паролем; пароль отправьте в другом канале (например, по телефону).
3. **Telegram/Signal** — отправьте сообщение в режиме «самоуничтожения».

> Не пересылайте `sk_*` и `whsec_*` обычной электронной почтой или в открытых чатах.

---

## 7. Что будет дальше

1. Мы добавим полученные ключи в безопасное хранилище проекта.
2. Подключим ваши Price ID к логике оплаты.
3. После тестов пришлём инструкции, как переключиться в «Live mode» (для реальных платежей).
